# 481. Magical String

## Intuition

The magical string `s` is generated by observing its own elements. The sequence starts with `"122"`. We can use two pointers (or indices) to dynamically generate the string:
- `head`: Points to the current number which dictates the length of the next group of identical numbers.
- `tail`: Points to the position where the next numbers will be placed.

Since we start with `"122"`, `head` will initially point to index 2 (value `2`), meaning the next group should have length 2. The `tail` is at index 3, and since the last added number was `2`, the next group must consist of `1`s.

## Algorithm

1. **Base Cases**: If `n <= 0`, return `0`. If `n <= 3`, return `1` (since the first 3 characters are `"122"`, which contains exactly one `'1'`).
2. **Initialization**: Create an array `a` of size `n + 1` to store the generated sequence. Initialize the first three elements as `a[0] = 1`, `a[1] = 2`, `a[2] = 2`.
3. Set `head = 2` and `tail = 3`. Set the next number to be generated `num = 1`. Initialize `count = 1` to track the number of `1`s (since `a[0]` is `1`).
4. **Generation**: Loop while `tail < n`:
    - The value `a[head]` tells us how many times to append `num`.
    - Iterate `a[head]` times:
        - Set `a[tail] = num`.
        - If `num == 1` and `tail < n`, increment `count`.
        - Increment `tail`.
    - Switch `num` from `1` to `2` or from `2` to `1` (can be done using `num = 3 - num` or bitwise XOR `num = num ^ 3`).
    - Increment `head` to read the next group length.
5. Return the `count` of `1`s.

## Complexity Analysis

### Time Complexity: $\mathcal{O}(n)$
- We generate the array up to index `n`. The while loop advances `tail` by 1 or 2 at each step, taking $\mathcal{O}(n)$ total operations.

### Space Complexity: $\mathcal{O}(n)$
- We use an array of size `n + 1` to store the sequence, which requires $\mathcal{O}(n)$ additional space.
